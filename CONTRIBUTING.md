# 🤝 Contribution Guide

We'd love to have you contribute to this project! Please follow the steps below.

---

## How to Contribute?

### 1. Fork the Repository
Go to GitHub and click the **Fork** button at the top.

### 2. Clone Locally
```bash
git clone https://github.com/rashid-mamun/devops-linux-cheatsheet.git
cd devops-linux-cheatsheet
```

### 3. Create a New Branch
```bash
git checkout -b add/new-command-section
```

### 4. Make Changes
Edit the `docs/cheatsheet.md` file.

### 5. Commit and Push
```bash
git add -A
git commit -m "add: kubernetes helm commands section"
git push origin add/new-command-section
```

### 6. Open a Pull Request
Go to GitHub and click the **Compare & pull request** button.

---

## What Types of Contributions Are Welcome?

- ✅ Adding new commands
- ✅ Fixing incorrect explanations
- ✅ Adding new real-world examples
- ✅ Fixing typos
- ✅ Adding new sections (e.g., Helm, Terraform)

---

## What Format Should I Use?

Write each command in the following format:

````markdown
### `command-name` — What it does in one line

```bash
command --flag         # Explanation in Bangla
command --other-flag   # Explanation in Bangla
```

**Explanation:** Write a detailed explanation in Bangla. Keep technical terms in English.
````
